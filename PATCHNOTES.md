# Patch Notes

## v0.1.1 — 2026-04-15

### analyze button now only processes new prompts since last run
Fixed a bug where clicking Analyze would re-send all of today's logged prompts to the LLM on every run, even entries already analyzed in a previous session. A new `analyzedAt` timestamp is now stored in the day's weekly record, and subsequent runs filter entries to only those logged after that timestamp.

### add JSON-RPC dispatch integration tests for MCP server
Extended the MCP test suite with two new integration tests covering JSON-RPC dispatching end-to-end. Ensures the MCP server's tool-call routing is exercised under realistic conditions, not just in unit isolation.

### add promptiq mcp command exposing stdio MCP server
Introduced a new `promptiq mcp` CLI subcommand that starts a stdio-based MCP server, exposing PromptIQ's analysis capabilities to MCP-compatible clients such as Claude Desktop. Includes `get_day_analysis`, `get_week_summary`, `list_analyzed_dates`, and `get_tip_correlation` tools.

### add sinceCount, last analysis time, and fix test mocks for analyze button
Added a "prompts since last analysis" counter to the Status tab and wired up the last-analysis-time display in the UI. Also fixed broken test mocks that were causing false-positive passes in the server-analyzer test suite.

### add classifier-context-hardening pipeline learnings
Recorded learnings from the classifier context-hardening pipeline run into `learnings.md`. No code changes.

### add feedback --acted command and tip correlation tracking
Added a `promptiq feedback --acted` subcommand so users can mark that they acted on a daily improvement tip. The server now computes and exposes a correlation score between acting on tips and prompt quality improvements over time.

### add Analyze button to Status tab for on-demand analysis
Added an "Analyze" button to the Status dashboard tab. Clicking it POSTs to `/api/run-analysis`, which performs a catchup pass for any missed days and then runs today's analysis — giving users a manual trigger without needing the CLI.

### add slash-command classifier and remove prompt truncation
Extended the classifier to recognise Claude Code slash-commands and exclude them from analysis alongside other control prompts. Also removed the earlier prompt-truncation logic, letting the full prompt text reach the LLM.

### add missing computeDiff tests and integration test for spot analyzer
Added unit tests for `computeDiff` in `diff-util.ts` and a new integration test covering the spot-analyzer endpoint (`/api/analyze-prompt`) end-to-end. Closes a gap where the diff helper had no test coverage.

### add Analyzer tab for on-demand single-prompt analysis
Introduced a new "Analyzer" tab in the dashboard. Users can paste any prompt and receive a rubric-scored breakdown with criterion scores, detected patterns, improvement suggestions, and a rewritten version — all powered by the spot analyzer.

### add production-ready README with 12-factor agents coverage
Rewrote the README from a stub into a full production guide covering installation, configuration, CLI commands, the web dashboard, the MCP server, and the 12-factor agent design principles applied throughout the project.

### bump max_tokens to 8000 and switch to Nunito font in dashboard
Increased the Claude API `max_tokens` ceiling from the default to 8 000 to reduce truncated analysis responses. Switched the dashboard font to Nunito for a cleaner look.

### add mainTip to DayAnalysis fixtures in drm.test.ts
Added the missing `mainTip` field to DRM test fixtures so they match the current `DayAnalysis` type, resolving type errors that were masking test failures.

### add mainTip field to day analysis — single most impactful improvement tip
Extended `DayAnalysis` and `WeekDayRecord` with a `mainTip` field containing the single most actionable improvement tip for the day. The tip is generated by the LLM alongside the existing scores and patterns and surfaced prominently in the dashboard.

### classify and exclude control prompts from analysis
Added a classifier module that identifies non-task prompts (e.g. greetings, meta-questions, short filler input) and excludes them before scoring. This prevents control prompts from diluting daily averages and pattern detection.

### correct five data/UX bugs (W1–W5)
Fixed five bugs identified in a post-launch audit: cross-week `previousDayScore` fallback, Monday/Sunday boundary handling in the renderer, a stale-UI issue after analysis, missing field guards in the Patterns tab, and an incorrect week-label edge case.

### add W2 Monday→Sunday cross-week previousDayScore fallback test
Added a targeted regression test for the W2 bug where fetching the previous day's score on a Monday would fail to cross the week boundary into Sunday's record.

### clarify tool schema descriptions and restore max_tokens to 8192
Improved the descriptions in the `report_analysis` tool schema so the LLM produces more reliable structured output. Also restored `max_tokens` to 8 192 after an earlier accidental reduction.

### reduce analyze token cost (sonnet, no history injection, truncation, weekly synthesis)
Cut per-analysis token usage by switching to claude-sonnet, removing historical context injection from `analyzeToday`, adding prompt truncation, and moving historical synthesis to the weekly compression step. Reduces cost significantly for users with large daily prompt volumes.

### add individual day rows with detail panel to Patterns tab
The Patterns tab now lists each analyzed day as an expandable row. Clicking a day reveals its top patterns, suggestions, and the main tip, giving users a browsable history without leaving the tab.

### repair broken onclick on Patterns rows
Fixed a JavaScript quoting error that caused the Patterns tab row `onclick` handlers to throw a syntax error, making the detail panels unopenable.

### add clickable detail panels to Patterns tab
Initial implementation of expandable day rows in the Patterns tab. Each row can be clicked to reveal the full pattern breakdown for that day.

### add web UI dashboard, catchup analysis, and startup scheduling
Launched the web dashboard server (`promptiq serve`) with Status, Patterns, and Weekly tabs. Added a catchup analysis command to process days missed since the last run, and wired up automatic scheduling so the server triggers analysis at the configured time each day.

### merge re-audit into original audit file, remove duplicate
Consolidated two overlapping audit documents into a single file in the pipeline artifacts directory. No code changes.

### add pipeline artifacts (retros, working logs) to project root
Committed the implementation pipeline's working logs, retrospectives, and audit documents to the repository root for future reference.

### implement 12-factor agent improvements (F3, F4, F9, F11)
Applied four of the 12-factor agent principles: structured tool output (F3), idempotent analysis runs (F4), explicit rubric-driven evaluation (F9), and separation of retrieval from generation (F11). These changes make the analysis pipeline more reliable and easier to test.

### initial PromptIQ implementation
First working version of PromptIQ: CLI logging (`promptiq log`), daily analysis (`promptiq analyze`), a data retention model (DRM) for weekly/monthly rollup, and a rubric-based LLM scorer. Establishes the core data flow from raw prompt capture through scored day records.
